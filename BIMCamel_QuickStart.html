BIMCamel IFC Exporter — Quick Start

Fast, free, open-source Navisworks → IFC exporter (IFC4 & IFC2x3)

What it does

Exports your Autodesk Navisworks model to IFC (IFC4 or IFC2x3). Navisworks has no native IFC export; BIMCamel adds it with a fast, memory-bounded engine and a zero-config first-click export.

Requirements

Autodesk Navisworks Manage or Simulate 2024, 2025, 2026 or 2027 (Win64), with .NET Framework 4.8 (installed alongside Navisworks).

After install

Restart Navisworks. A BIMCamel tab appears on the ribbon with two buttons: IFC exporter and About. If the tab is missing, open Tools / Plug-ins to confirm BIMCamel loaded for your Navisworks version.

Export in 3 steps

  1. BIMCamel tab → IFC exporter.
  2. Pick a schema (IFC4 by default) and scope (whole model by default). Everything else has sensible defaults.
  3. Click Export IFC, choose a path. Done.

Useful options

  • Scope: whole model, current selection, active section box, a saved/search set, or batch (multiple sets → one IFC each).
  • Quality presets: Small file / Balanced / High detail.
  • Object → IFC class mapping; property-set mapping & dedup; base quantities (volume / area / length); multi-storey spatial structure from a Level property.
  • Georeferencing: base-point modes with live preview (IFC4 IfcMapConversion; IFC2x3 baked placement).
  • Size splitting: cap the output size; large exports roll into name_001.ifc, name_002.ifc, … each a complete, standalone IFC.
  • Export profiles: save / load all settings as JSON.
Performance tip: Deactivate any active DataTools / external-database link (Home → DataTools) before exporting — it runs one query per object and can add many minutes to a large export.
